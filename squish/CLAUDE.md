## 行为准则

**Tradeoff:** 以下准则偏向谨慎而非速度。对简单任务可灵活判断。

### 1. 先想后写

- 不要假设，不要隐藏困惑，公开权衡取舍
- 实现前先陈述假设；如有不确定，询问
- 如有更简单的方案，指出
- 如有不清楚，停下来，说出困惑点，提问

### 2. 简洁优先

- 最少代码解决问题，不做推测性开发
- 不写未被要求的功能
- 不为单次使用的代码建抽象层
- 不写不可能发生场景的错误处理
- 如果 200 行能压缩到 50 行，重写

### 3. 精准改动

- 只碰必须改的，只清理自己造成的凌乱
- 不改动相邻代码、注释、格式
- 不重构没坏的东西
- 修改造成孤立的导入/变量/函数 — 清理掉

### 4. 目标驱动 + 小步验证

- 每次只实现一个功能，验证通过后再做下一个
- 每完成一个步骤，在开发日志中记录
- 遇到需要用户操作的事（注册账号等），明确告知步骤，不代劳

```markdown

## 今日完成
- [完成项 1]
- [完成项 2]

- [待办 1]
```

## 关键约束

1. **零预算**：所有服务必须用免费层
2. **纯浏览器端**：核心计算在客户端完成，不依赖付费 API
3. **英文界面**：目标用户是海外英语用户
4. **渐进开发**：每次只做一步，验证后再继续
5. **用户操作用户做**：注册账号、配置服务等由用户自己完成，AI 负责代码
